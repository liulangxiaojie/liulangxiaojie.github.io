AngularJS表达式

AngularJS 是一个JavaScript框架，通过script标签来引入到HTML页面。

angular表达式双大括号内{{ }}，{{ }}是体现angularJS的重要特性。
{{ }}可以包含文字、运算符和变量,支持三元表达式。但是不支持条件判断、循环及异常；
{{ }}中放入’’表示字符串

AngularJS事件

ng-click

定义了指令定义了 AngularJS 点击事件。
开关灯思想通过这个小例子：
核心思想是通过改变flag的值来操作元素的显示和隐藏
ng-show / ng-hide 第一次初始化时，消耗性能较高; ng-if 当频繁切换时，消耗性能比较高

<!DOCTYPE html>
<html ng-app="module">//确定了angular的作用范围，后边的module是模块的名字
<head>
    <meta charset="UTF-8">
    <title>开关灯</title>
</head>
<body>
<div ng-controller="myCtrl">//控制器的作用范围，后边写的是控制器的名字
    <div ng-click="show()">{{flag?'打开':'隐藏'}}</div>
    //ng-click后边的一定是一个方法，()绝对不能少
    <div ng-hide="flag">内容</div>
</div>
</body>
<script src="angular.js"></script>
<script>
    var app=angular.module('module',[]);
    //创建一个模块
    app.controller('myCtrl', function ($scope) {
        //创建一个控制器，第一个是控制器的名字
        $scope.flag=true;//先在$scope上定义flag的值
        $scope.show= function () {
            $scope.flag=!$scope.flag;
            //每次点击都会改变flag的值
        }
    })
</script>
</html>
  • ng-hide控制DOM元素的隐藏和显示，值为true，隐藏；值为false，显示。操作css样式，不会产生作用域
  • ng-show控制DOM元素的显示和隐藏，值为true，显示；值为false，隐藏。操作css样式，不会产生作用域
  • ng-if控制DOM元素的存在与不存在，当值为true的时候，DOM元素存在，false的时候DOM元素不存在。频繁在操作DOM元素，会产生作用域，不建议使用

关于6个this指向的总结

1、自执行函数中的this永远都是window。

2、当函数被调用的时候，看前面是否有”.”,”.”前面是谁，this就是谁。

3、当触发一个元素身上的事件，执行对应的函数，函数中的this指向当前这个元素。

4、在构造函数中，this指向当前实例。

5、call(),apply(),bind()改变this的指向。

6、回调函数中this一般指向window。

回调函数是什么?

  • 就是一个函数A，作为参数，传入了另一个函数B中，然后被B在某个时间调用。

使用git发布一个项目

1、创建一个新的仓库

  • 在写完仓库的描述后， 勾选 格式化 Initialize this repository with a README ,创建一个使用说明。

2、发布页面

  • 创建好仓库后，在设置项–> Options 选项中 勾选 launch automatic page generator进入到博客的页面后继续发布 点击 Continue to Layouts –>选择模板后 点击 Public page发布页面。

3、修改仓库的默认分支，然后跟新

  • 发布页面完成后 点击 Setting 在 Branches 里边把仓库的默认分支更改 master 为 gh-pages 然后 点击 updata完成跟新。

4、把gh-pages分支下的内容克隆到本地

  • 把gitHub远程仓库下托管的源文件下载到了本地
  • 让本地的文件项目和远程仓库保持了一个链接的通道
  • 下载到本地文件夹里边的【.git】隐藏文件夹不要删除

5、本地仓库和远程仓库关联起来【发布项目非必选可以省略】

6、最新文件上传

  • 修改本地的项目文件，把最新的项目文件上传到gh-pages这个分支下。

7、项目展示的地址

  • 回到你发布项目的仓库，点击Settings,在GitHub Pages就能看到项目展示的地址。

8、移动端项目二维码生成

  • 在线生成地址：http://cli.im/
