关于6个this指向的总结

1、自执行函数中的this永远都是window。

2、当函数被调用的时候，看前面是否有”.”,”.”前面是谁，this就是谁。

3、当触发一个元素身上的事件，执行对应的函数，函数中的this指向当前这个元素。

4、在构造函数中，this指向当前实例。

5、call(),apply(),bind()改变this的指向。

6、回调函数中this一般指向window。

回调函数是什么?

方法一

1、创建一个新的仓库

  • 在写完仓库的描述后， 勾选 格式化 Initialize this repository with a README ,创建一个使用说明。

2、发布页面

  • 创建好仓库后，在设置项–> Options 选项中 勾选 launch automatic page generator进入到博客的页面后继续发布 点击 Continue to Layouts –>选择模板后 点击 Public page发布页面。

3、修改仓库的默认分支，然后跟新

  • 发布页面完成后 点击 Setting 在 Branches 里边把仓库的默认分支更改 master 为 gh-pages 然后 点击 updata完成跟新。

4、把gh-pages分支下的内容克隆到本地

  • 把gitHub远程仓库下托管的源文件下载到了本地
  • 让本地的文件项目和远程仓库保持了一个链接的通道
  • 下载到本地文件夹里边的【.git】隐藏文件夹不要删除

5、本地仓库和远程仓库关联起来【发布项目非必选可以省略】

6、最新文件上传

  • 修改本地的项目文件，把最新的项目文件上传到gh-pages这个分支下。

7、项目展示的地址

  • 回到你发布项目的仓库，点击Settings,在GitHub Pages就能看到项目展示的地址。

数组常用方法

1.push

  • 向数组的末尾添加一个元素，改变原先数组的长度

2.unshift

  • 向数组的开头添加一个元素，改变原先数组的长度

3.shift

  • 数组的第一个元素从其中删除，并返回第一个元素的值

4.pop

  • 数组的最后一个元素从其中删除，并返回第一个元素的值

5.splice

  • 从索引n开始，删除m个； 返回值：被删除的数据以一个新数组的形式返回；原数组发生改变；
  • 从索引n开始，删除0个；添加x的新内容,添加到索引n前边; 返回值：空数组； 原数组发生改变
  • 从索引n开始(包括n)，删除m个，替换成x新内容；返回值：被删除的数据以一个新数组的形式返回；原数组发生改变；

6.slice

  • 查找索引n到m，但是不包含m,不改变原数组

7.concat

  • 数组拼接

8.sort ( function ( a,b ) {return a-b } )数组排序

  • return a-b为升序，return b-a为降序;原数组发生改变

9.reverse

  • 数组翻转,原数组发生改变

10.toString

  • 把数组转成字符串； 返回值：以逗号分隔的字符串； 原数组没发生改变；

11.join ( ‘字符’)

  • 把数组转成任意形式拼接的字符串； 返回值：任意形式拼接的字符串； 原数组没有发生改变；

12.indexOf ( )

  • 查找对应的内容，没找到对应的内容，返回-1； 找到，返回对应内容的索引；

13.find

  • 符合条件时，返回值是找到的那一项
  • 不符合条件时，返回值是undefined.

14.filter

  • filter遍历数组中的每一项，把符合条件的挑选出来，作为一个新数组返回，不会改边原数组。

15.forEach

  • 遍历数组中的每一项，里面是个匿名函数；这个匿名函数中有两个参数：item（数组中的每一项），index（数组中每一项的索引）；他没有返回值；

16.map

  • 功能和forEach()一样，但是他有返回值；
